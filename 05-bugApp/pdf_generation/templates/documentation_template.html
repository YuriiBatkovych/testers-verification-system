Ecommerce Shop Manual

Introduction

Basic information

The proposed e-commerce shop (hereinafter referred to as the shop) available for testing offers the possibility to purchase goods in 4 categories:

  • Books
  • Coffee mugs
  • Luggage tags
  • Mouse Pads

Selected items can be ordered to a limited number of countries, the list of which should be visible during the checkout screen. These countries are: Brazil, Canada, Germany, Turkey, India, United States. The mentioned countries are fully supported, meaning that orders can be shipped to any part of the specified countries.

Each user of the online store is assigned one specific role out of three: USER, STAFF, ADMIN. An unregistered user is treated as having the USER role. Details about the permissions and capabilities of users with each role will be described in the following chapters.

Note

Note: As soon as you notice any bug while testing the system, immediately click the red "Declare bug" button located in the left panel of the screen. This will help us understand that you have noticed incorrect behavior.

User Authentication

Login

Upon entering the main page of the online store, a user who wishes to log in or create an account should click the "Login" button located at the top of the page.

In the opened window:

  • If a person has an account in the system, they should log in by entering their email and password in the appropriate fields or by using a Google account, by clicking the appropriate button, which will direct the user to the authentication procedure provided by Google.
  • If the user does not have an account in the system of the tested shop, they should click on the "Sign up" button and create an account in the shop's domain by entering their email and a chosen password in the appropriate fields or by choosing the option to create an account using Google. Each new user is assigned the USER role.

Browsing Products

On the main store page, products are presented by categories. The following basic information about the product should be displayed there:

  1. Name
  2. Sample product image
  3. Price

Under each product, there is a button "Add to cart" which adds the product to the cart with a quantity of 1 piece.

By default, 10 products can be viewed on one page. The user can change this number by selecting the desired quantity from the dropdown labeled "Page size". Possible options are 5, 10, 20, 50.

To navigate to the next page of products, the user should click on the page number or arrow in the list of available pages at the bottom of the screen.

In the left part of the page, there is a list of current product categories. Currently, the store has 4 categories of products:

  • Books
  • Coffee mugs
  • Luggage tags
  • Mouse Pads

The default category for browsing products is books. To change the category, the user should click on the name of the desired category on the list in the left sidebar.

Additionally, there is an option to view detailed information about a selected product. To do this, the user should click on the image or name of the product of interest. It should open a page containing a detailed description of the product.

The page with a detailed product description also includes an "Add to cart" button, allowing the user to add the product to the cart. This button is located at the bottom of the screen.

Searching Products

To search for a desired product, the user should enter a known part of the product name in the search panel located at the top of the screen and labeled with the text "Search for products …", then click the "Search" button or press Enter.

If there are any products with names that partially match the entered text, a list of these products should be displayed on the screen. Otherwise, the user should see a message informing them that such products do not exist.

Order History

The option to check the order history is only available for registered and logged-in users.

To view the order history, a logged-in user, while on the main store page, should click the "Orders" button. After that, a page containing basic information about all the orders placed by the user will be opened.

The following fields should be displayed precisely in the form of table columns:

  1. Order Tracking Number
  2. Total Price
  3. Total Quantity
  4. Order Placement Date

If the user has not placed any orders, upon accessing the Orders page, a message indicating the absence of orders will be displayed.

Members Page

The Members page allows users to check discounts and/or promotions available to a specific user. This option is available only for registered and logged-in users.

By clicking the "Members" button, the user is redirected to a page with information about the discounts granted to them or their absence.

Discounts are granted as follows:

  • Every new user receives a 10% discount.
  • For every 5 orders (except every 10th order), the user receives a 30% discount. This means the user will have a 30% discount on the 5th, 15th, 25th order, and so on.
  • For every 10 orders, the user receives a 50% discount. This means the user will have a 50% discount on the 10th, 20th, 30th order, and so on.

Cart Details

In the top right corner of the screen, the user can see the number of products currently in their cart and their total value in US dollars.

Important: If any discount is available for the user, it is automatically applied to the current order. Consequently, the total value of products in the user's cart will be reduced by the percentage of the available discount. This reduced value will be displayed in the top right corner of the screen.

To view detailed information about the products currently in the cart, the user should click on the shopping cart icon located in the top right corner of the store interface.

Upon clicking the cart icon, a page will be displayed containing the following information in the form of a table:

  1. Product Image
  2. Name and price of each item
  3. Quantity of specific items in the cart

The Cart Details interface allows users to remove products from the cart, increase or decrease the quantity of specific items in the cart.

Additionally, information about the total quantity of items in the cart, the total cost of items, and the shipping cost is displayed on a separate row for the user.

If a discount is applied to the order, information about it will also be displayed.

Completing an Order

To complete an order, the user should go to the Checkout page by clicking the Checkout button located in the bottom right corner of the Cart Details page, fill in all the required fields, and click the Purchase button.

All fields on the Checkout page are required to be filled, with:

  • The customer's email automatically filled in for registered users, with the option to change its value
  • The Expiration Month and Expiration Year of the credit card automatically filled with the current year and month, with the option to change

Restrictions on some fields:

  • The email field value should be a valid email, which must be checked with appropriate regular expressions in the code
  • The values of the First name field should be at least {{defaultFirstNameMinLength}} characters long
  • The values of the Last name field should be at least 2 characters long
  • The values of the Street fields should be at least {{defaultShippingAddressStreetMinLength}} characters long
  • The values of the City, and Zip code fields should be at least 2 characters long
  • The Country field value is selected from the list of available countries. These are the following countries: Brazil, Canada, Germany, Turkey, India, United States.
  • The selection of the State field value is only possible after choosing the Country. State is selected from the list of regions of the respective country
  • The Cart type field value is selected from two available options: Mastercard and Visa
  • The Name on card field value should be at least 2 characters long
  • The values of the Security code and Card number fields should contain only numeric characters
  • The Security code field value should have 3 characters
  • The Card number field value should have 16 characters

If the restrictions for the fields are not met, the user will be shown an appropriate message.

Checking the checkbox labeled "Billing Address same as Shipping Address" automatically fills the Billing Address section with values identical to the values of the fields in the Shipping Address section

Adding, Editing, and Deleting Products

Adding a New Product

The functionality of adding a new product is available only to users with ADMIN or STAFF roles.

To add a new product, the user should click on the "Add new product" button located in the left panel of the screen, fill in all the required fields in the displayed form, and confirm the form by clicking the "Add" button.

All fields on the new product adding page are required to be filled, with:

  • The product category being selected from a dropdown list containing names of all categories currently available in the system.
  • The Unit price and Units in stock fields are automatically filled with zeros.
  • The Active field is set to False by default.

Restrictions on some fields:

  • The values of the Name field should be at least {{defaultProductNameMinLength}} characters long.
  • The values of the Sku, Image URL, and Description fields should be at least 2 characters long.
  • The value of the Units in stock field should be a natural number.
  • The value of the Unit price field should be a number greater than or equal to zero.

If the restrictions for the fields are not met, the user will be shown an appropriate message. In case of failure to add a new product, the user will be shown an appropriate message.

Editing an Existing Product

The functionality of editing an existing product is available only to users with ADMIN or STAFF roles.

To update information about an existing product, the user should navigate to the page containing detailed information about that product, click on the "Update product" button, fill in all the required fields in the product editing form, and then confirm the form by clicking the "Edit" button.

All fields in the product editing form are automatically filled with the current values for that product, with the option to change them. The editing form is identical to the adding form. Therefore, the requirements and restrictions for the fields in the editing form repeat the requirements for the fields in the adding form.

If the restrictions for the fields are not met, the user will be shown an appropriate message. In case of failure to update the product information, the user will be shown an appropriate message.

Deleting an Existing Product

The functionality of deleting an existing product is available only to users with ADMIN or STAFF roles.

To delete an existing product, the user should navigate to the page containing detailed information about that product and click on the "Delete product" button.

In case of failure to delete the product, the user will be shown an appropriate message.

Adding, Editing, and Deleting Categories

Adding a New Category

The functionality of adding a new category is available only to users with ADMIN or STAFF roles.

To add a new category, the user should click the "Add new category" button located in the left panel of the screen, enter the name of the new category in the appropriate field of the form, and confirm the form by clicking the "Add" button.

The form for adding a new category contains only one field, "New Category name," with:

  • It is required to be filled.
  • The value of the field should contain at least 2 non-empty characters.
  • The value of the field should be different from the names of other product categories already existing in the system.

If the restrictions for the field are not met, the user will be shown an appropriate message. In case of failure to add a new category, the user will be shown an appropriate message.

Editing an Existing Category

The functionality of editing a category is available only to users with ADMIN or STAFF roles.

To edit categories, the user should click the "Edit category" button located in the left panel of the screen, fill in the required fields of the form, and confirm the form by clicking the "Edit" button.

The form for editing a category contains two fields, both of which are mandatory, with:

  • The "Current Category Name" field is filled by choosing from a dropdown list containing names of all product categories currently supported by the system.
  • The "New Category name" field should contain at least 2 non-empty characters, and its value should not repeat the name of existing categories.

If the restrictions for the field are not met, the user will be shown an appropriate message. In case of failure to edit the category, the user will be shown an appropriate message.

Deleting an Existing Category

The functionality of deleting a category is available only to users with ADMIN or STAFF roles.

To delete categories, the user should click the "Delete category" button located in the left panel of the screen, select the category to delete, and confirm by clicking the "Delete" button.

Note: Deleting a category will also delete all products in that category.

The form for deleting a category contains only one field, "Current Category name," with:

  • The field is mandatory.
  • The field is filled by choosing from a dropdown list containing names of all product categories currently supported by the system.

If the restrictions for the field are not met, the user will be shown an appropriate message. In case of failure to delete the category, the user will be shown an appropriate message.

User Management

User Management Overview

The functionality of user management is available only to users with an ADMIN role.

To access the user management panel, the admin should click the "User management" button located in the left panel of the screen.

On the displayed page, a full list of all system users will be shown along with the following information about them:

  • First Name
  • Last Name
  • Email Address
  • Role

Email address and role are essential pieces of information for each user.

Adding a User

To add a user, the admin should click the "Add" button located at the bottom of the account management panel, fill in the user addition form, and confirm it by clicking the "Add" button at the bottom of the form.

The user addition form contains 4 fields: First name, Last name, Email, and Role, with:

  • The email field is mandatory.
  • The value of the email field must be a valid email address, which is checked using appropriate regular expressions.
  • Values of the First name and Last name fields should be at least 2 characters long.
  • The Role field is filled by choosing from a dropdown list containing all possible roles currently supported in the system.

If the restrictions for the field are not met, the user will be shown an appropriate message. In case of failure to add a user, the user will be shown an appropriate message.

Editing a User

To update user information, the admin should click the "Update" button located in the respective row of the user table, fill in the user editing form, and confirm it by clicking the "Edit" button at the bottom of the form.

The user editing form contains 4 fields: First name, Last name, Email, and Role. The form is automatically filled with the current user data with the option to change them. Additionally:

  • The email field is mandatory.
  • The value of the email field must be a valid email address, which is checked using appropriate regular expressions.
  • Values of the First name and Last name fields should be at least 2 characters long.
  • The Role field is filled by choosing from a dropdown list containing all possible roles currently supported in the system.

If the restrictions for the field are not met, the user will be shown an appropriate message. In case of failure to edit a user, the user will be shown an appropriate message.

Deleting a User

To delete a user, the admin should click the "Delete" button located in the respective row of the user table.

Note: Deleting a user will permanently remove their data from the system, and this action cannot be undone.

In case of failure to delete a user, the user will be shown an appropriate message.
